# Solution Note: Multi-Region Product Sum with OR Logic

The task required summing amounts from the **Data** sheet based on product and region selections, where users could select up to 3 regions or "All" regions.

## Data Structure
- **Data sheet**: Headers in row 1 (Product, Region, M1)
  - Column A: Product names
  - Column B: Region names  
  - Column C: M1 amounts (numeric values)
  - Data started in row 2

- **Dashboard sheet**: 
  - Cell C2: Product selection (e.g., "AAA")
  - Column Q (rows 1-5): Region selections ("All", "EMEA", "LAC", "APAC", "US")
  - Cell C4: Target cell for the calculated outcome

## Python/openpyxl Approach
The solution read all data from the **Data** sheet starting at row 2:
1. Extracted the product value from **Dashboard!C2**
2. Collected region selections from **Dashboard!Q1:Q5** (up to 3 regions plus "All" option)
3. Filtered **Data** sheet rows where column A matched the selected product
4. Applied OR logic: if "All" was selected, summed all matching product rows; otherwise summed only rows where column B matched any of the selected regions
5. Wrote the calculated sum to **Dashboard!C4**

This approach handled the multi-criteria OR logic that SUMIFS cannot natively support, providing flexibility for dynamic region selection.
