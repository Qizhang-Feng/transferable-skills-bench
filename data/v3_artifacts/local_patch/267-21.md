# Solution Note: Sheet Merging with Dynamic Lookup

The task required creating a comprehensive report in the 'merging' sheet that compiles data from multiple monthly sheets (SH1, RP1, etc.) with automatic updates as new monthly data is added.

**Data Structure:**
- Source sheets: 'SH1' (Month 1) and 'RP1' (Month 2), each containing columns 'ITEM', 'ID', and 'QTY' starting from row 1 (header) with data from row 2 onwards
- Target sheet: 'merging' with columns 'ITEM', 'ID', 'MONTH1', 'MONTH2' (header in row 1)
- Output range: C2:D11 in the 'merging' sheet

**Python/openpyxl Approach:**
1. Read the 'ID' column (column B) from the 'merging' sheet starting at row 2 to identify which items to look up
2. For each ID in 'merging' sheet, search for matching IDs in column B of 'SH1' sheet (rows 2-6)
3. When a match was found, retrieve the corresponding 'QTY' value from column C of 'SH1' and write it to column C ('MONTH1') of the 'merging' sheet
4. Repeat the same lookup process for 'RP1' sheet, writing matched 'QTY' values to column D ('MONTH2')
5. If no match was found for an ID in a particular monthly sheet, the cell remained empty or was set to 0/blank

This approach used VLOOKUP-style logic via Python dictionaries, mapping IDs to quantities for efficient lookup across multiple source sheets.
