# Solution Note: INDEX-MATCH Formula for Multi-Row Lookup

## Problem Context
The user needed to retrieve values from the 'DATA' sheet into cells C15:G16 on the 'INPUTS' sheet using INDEX-MATCH formulas. The first row (C15:G15) returned correct values, but the second row (C15:G16) failed to populate.

## Spreadsheet Structure
- **DATA sheet**: Contains assumption data starting at row 3, with labels in column A and values in column B. The location type (Metro/Regional) was in column E, rows 4-5.
- **INPUTS sheet**: Target cells were C15:G16, requiring a 2-row by 5-column output range.
- **Sheet2**: Contains regional expense data with categories in column B and year values across columns C-G (years 1-5).

## Solution Approach
The task required a two-dimensional lookup that matched both:
1. Row criteria (expense category from INPUTS sheet)
2. Column criteria (year number from INPUTS sheet row 2)

The Python/openpyxl solution read:
- Sheet2 column A for location type filtering (Regional)
- Sheet2 column B for expense categories (Advertising & Marketing, Electricity, Insurance, Land Tax)
- Sheet2 columns C through G for year 1-5 values
- INPUTS sheet row 3 (columns C-G) for year headers

The logic matched the expense category and year, then wrote the corresponding values to INPUTS C15:G16. A standard INDEX-MATCH formula cannot handle this two-dimensional array lookup without array formulas or helper columns; Python provided direct cell-by-cell mapping.
