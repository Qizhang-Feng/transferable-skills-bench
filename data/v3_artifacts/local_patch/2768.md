# Solution Note: Inventory Allocation by Order Priority

The task required allocating available inventory to order lines in Sheet1 based on Order No priority, with results written to column H (rows 2-18).

**Data Structure:**
- Sheet1 contained order lines with headers in row 1, data starting row 2
  - Column A: 'Order No' (e.g., 1001-A, 1002-B)
  - Column D: 'Item code' 
  - Column G: 'Order Qty' (requested quantity)
  - Column H: Target column for allocated quantities

- Sheet2 contained incoming inventory with headers in row 1, data starting row 2
  - Column A: 'Order No'
  - Column B: 'Item code'
  - Column D: 'Incoming Qty'

**Logic Applied:**
The Python/openpyxl approach involved:
1. Reading all rows from Sheet2 to build an inventory dictionary, aggregating 'Incoming Qty' (column D) by the combination of 'Order No' (column A) and 'Item code' (column B)
2. Iterating through Sheet1 rows sequentially (maintaining order priority)
3. For each row, matching 'Order No' (column A) and 'Item code' (column D) against available inventory
4. Allocating the minimum of 'Order Qty' (column G) and remaining inventory
5. Deducting allocated quantity from inventory tracker
6. Writing allocated quantity to column H of the current row
7. Writing 0 when inventory was exhausted

This sequential processing ensured earlier order lines received priority allocation when multiple lines requested the same item under the same Order No.
