# Solution Note: Prioritized Column Selection with #N/A Handling

## Task Summary
The task required creating a formula to populate column D ('Result') in the 'Query' sheet by selecting the first non-#N/A value from columns A, B, and C in each row, with left-to-right priority.

## Spreadsheet Structure
- **Sheet Name**: Query
- **Header Row**: Row 1 containing ['A', 'B', 'C', 'Result']
- **Data Range**: Rows 2-8
- **Input Columns**: A, B, C (columns A through C)
- **Output Column**: D ('Result')

## Python/openpyxl Approach
The solution read data from the 'Query' sheet starting at row 2 (first data row after header). For each row from 2 to 8:

1. Read the values from columns A, B, and C in sequence
2. Check if each cell contains the error value `#N/A` (represented as an `ErrorCell` object in openpyxl)
3. Applied priority logic: select column A value first; if A contains #N/A, check column B; if B also contains #N/A, check column C
4. If all three columns (A, B, C) contained #N/A, left column D blank (empty string)
5. Wrote the selected value (or empty string) to the corresponding cell in column D

The logic implemented a cascading fallback mechanism that prioritized leftmost non-error values, ensuring column D displayed the first available valid data point from the three source columns.
