# Solution Note: Filter Data Parsing and Row Expansion

## Task Overview
The data was located in the 'Input' sheet with headers in row 1 (wtype_id, name, status, filter) and data starting from row 2. The 'filter' column (column D) contained complex filter strings that needed to be parsed and split into separate columns in the 'Output' sheet.

## Approach
The solution read columns A through D from the 'Input' sheet. For each row, the filter string in column D was parsed using regular expressions to extract four key attributes:
- **orgId**: Extracted from patterns like `("orgId" = "VALUE")`
- **careType**: Extracted from patterns like `("careType" = "VALUE")`
- **specialtyId**: Extracted from `IN LIST` patterns, splitting comma-separated values
- **contextId**: Extracted from `IN LIST` patterns, splitting comma-separated values

When specialtyId or contextId contained multiple values in a list (e.g., "66,77" or "55,689,213"), the solution created a Cartesian product of all combinations, generating multiple output rows from a single input row. This explained why row 3 from Input (ge-QA1) expanded into rows 3-7 in Output.

The results were written to the 'Output' sheet starting at A1, with headers: wtype_id, name, status, orgId, careType, specialtyId, contextId (columns A through G). Empty cells were left blank, and leading zeros in numeric values were omitted as specified.
