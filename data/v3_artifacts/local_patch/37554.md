# Solution Note: Exact Match Filtering for TYPE Categories

The task involved fixing a SUMPRODUCT formula issue in sheet **Sheet1** where "Online Travel Agency" was incorrectly being counted as "Travel Agency" due to partial text matching.

## Spreadsheet Structure
- **Header row**: Row 1
- **Data start row**: Row 2
- **Source data**: Column A (TYPE) contained category names, Column B (Accounts) contained count values
- **Target output**: Column F (Accounts) in rows 2-17, with corresponding category names in Column D (TYPE)

## Python/openpyxl Approach
The solution read data from **column A (TYPE)** and **column B (Accounts)** starting from row 2 to capture all category-account pairs. The category list to match against was read from **column D (TYPE)** starting from row 2.

For each category in column D, the logic performed an **exact string match** (using equality comparison, not substring search) against all values in column A. When an exact match was found, the corresponding value from column B was summed. This ensured "Travel Agency" only matched "Travel Agency" exactly, not "Online Travel Agency".

The calculated sums were written to **column F (Accounts)** starting at row 2, aligning with each category name in column D. This exact-match approach eliminated the partial matching problem inherent in the SEARCH function.
