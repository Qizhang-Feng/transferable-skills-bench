# Solution Note: Transposing Lookup Values with Duplicates

The task involved retrieving data from Table 2 and transposing it horizontally into Table 1 on Sheet1, handling duplicate lookup values.

## Spreadsheet Structure

**Sheet1** contained two tables:
- **Table 1** started at row 6 with headers in columns C through H and beyond ("Column 1", "Return Value 1", "Return Value 2", etc.)
- **Table 2** started at row 6 with headers in columns L, M, N ("Column 2", "Value 1", "Value 2")

The header row was at **row 6**, with data starting from **row 7** onwards.

## Python/openpyxl Approach

The solution required:

1. **Reading Table 2** (columns L, M, N starting from row 7) to build a lookup dictionary where column L ("Column 2") contained the lookup keys, and columns M and N contained the associated values
2. **Reading Table 1** column C ("Column 1") starting from row 7 to identify the lookup values (which could be duplicates)
3. **Implementing logic** to handle duplicate keys by collecting all matching values from Table 2 for each lookup key
4. **Writing results** horizontally across columns D, E, F, G, etc. in Table 1, transposing the vertical data from Table 2 into horizontal rows
5. The target range for answers was **D8:D11**, indicating multiple rows of data needed processing

The key challenge was managing duplicate lookup values and ensuring each occurrence retrieved all corresponding values from Table 2, placed sequentially across the row.
