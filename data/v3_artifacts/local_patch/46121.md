# Solution Note: Monthly Transaction Allocation to Category Summary

## Task Overview
The task required reading transaction data from the 'Transactions' sheet and allocating totals to corresponding month-category cells on the '2022' sheet.

## Data Structure
**Source Sheet: 'Transactions'**
- Income section: Column A ('Month'), Column C ('Amount'), Column D ('Category')
- Data started at Row 3 (Row 2 was headers)
- Month values were text strings (e.g., "March")

**Target Sheet: '2022'**
- Row 1 and Row 3 contained month headers (January through December in columns B through M)
- Row 4 contained "Income" label
- Rows 5-8 contained category names in column A (e.g., "Chickens", "Turkeys", "Ducks", "Coturnix Quail")
- Target range was B5:M8 (12 months × 4 categories)

## Solution Approach
The solution iterated through each cell in the target range B5:M8. For each cell:
1. Determined the month from the column header (Row 3)
2. Determined the category from the row header (Column A)
3. Filtered 'Transactions' sheet data where Column A matched the month AND Column D matched the category
4. Summed the corresponding values from Column C ('Amount')
5. Wrote the sum to the target cell

This approach used SUMIFS-like logic implemented in Python with openpyxl to match multiple criteria and aggregate transaction amounts.
