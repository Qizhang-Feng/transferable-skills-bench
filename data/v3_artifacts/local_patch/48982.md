# Solution Note: Circular Reference in Check Number Formula

## Problem Analysis
The formula in cell B323 on Sheet1 was `=IF(A323="E","",MAX(B$7:B377)+1)`, which caused a circular reference error and displayed "000" instead of the expected maximum value plus one.

## Root Cause
The circular reference occurred because cell B323 was included within its own MAX range (B$7:B377). When Excel attempted to calculate the maximum value in this range, it needed to evaluate B323 first, which in turn needed the maximum value that included B323 itself—creating an infinite loop.

## Solution Approach Using Python/openpyxl

The task was solved by:

1. **Reading the data**: Accessed Sheet1 and read column B (Check #) from rows 7 to 377, excluding row 323 to break the circular dependency
2. **Finding the maximum**: Iterated through cells B7:B322 and B324:B377, filtering out empty cells and non-numeric values, then calculated the maximum numeric value
3. **Applying conditional logic**: Checked if cell A323 contained "E"; if true, left B323 empty, otherwise calculated MAX+1
4. **Writing the result**: Wrote the computed value directly to cell B323 as a static value rather than a formula

The header row was at row 5, with data starting from row 7. This approach eliminated the circular reference by computing the value programmatically instead of relying on a self-referencing formula.
