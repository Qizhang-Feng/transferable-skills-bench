# Solution Note: Deleting Rows with Floating-Point Precision Issues

## Problem Context
The data was in sheet **'final Recon Items'** with headers in row 1: 'Date', 'Branch', 'Ref', 'Amount'. Data rows started from row 2.

## Approach
The task required identifying and deleting groups of rows where column **'Amount'** (column D) contained values that should sum to zero, but due to floating-point precision errors, the exact sum was not precisely 0.00.

## Solution Method
1. **Read the data**: Loaded all values from column D ('Amount') in sheet 'final Recon Items', starting from row 2 (after the header row).

2. **Handle precision issues**: Instead of checking if the sum equals exactly zero, applied a tolerance-based comparison using a small epsilon value (e.g., 1e-9) to account for floating-point arithmetic errors. Values like -1.00000000093132E-02 and 0.0100000000020373 were treated as effectively summing to zero when their absolute sum was below the tolerance threshold.

3. **Identify rows to delete**: Grouped all data rows and calculated the sum of column D values. If the absolute value of the sum was less than the tolerance, marked these rows for deletion.

4. **Delete rows**: Removed the identified rows from the worksheet, preserving the header row.

5. **Output**: The remaining valid data was written back to the range **'final Recon Items'!A1:D5**.
