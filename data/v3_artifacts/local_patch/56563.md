# Solution Note: Summing Range with #NUM Error

The task required creating a SUM formula in cell C14 of sheet "AB" that would sum values in column "Amount" (column C) from rows 2 to 13, even when #NUM errors were present in the range.

## Spreadsheet Structure
- **Sheet Name**: AB
- **Header Row**: Row 1 contained column headers: '', 'Month', 'Amount'
- **Data Range**: Column C (Amount) from C2 to C13
- **Target Cell**: C14 for the formula result

## Python/openpyxl Approach

The solution involved reading data from sheet "AB", specifically column C (Amount) starting from row 2 through row 13. The Python script iterated through cells C2:C13 to extract numeric values while handling potential #NUM errors.

The logic applied was to check each cell's value type and error status. For cells containing numeric values, they were accumulated in a running sum. Cells containing errors (including #NUM) were skipped during the summation process.

After computing the total, the result was written to cell C14 on sheet "AB". The approach used openpyxl's cell value reading capabilities and conditional logic to filter out error values while preserving valid numeric data.

This method effectively replicated Excel's AGGREGATE or SUMIF functionality that ignores error values, ensuring the sum calculation completed successfully despite the presence of #NUM errors in the source range.
