# Solution Note: Four-Criteria INDEX/MATCH Formula Fix

The task involved creating a lookup formula with four matching criteria in Sheet1. The data table structure had:
- **Horizontal headers**: Row 2 contained quantity ranges (200-499, 500-999, etc.), Row 3 contained "qty code" values (1-6 repeated), and Row 4 contained "Grade" values (1-4 repeated)
- **Vertical headers**: Column A contained Grade values (125C125C, 125T125T, etc.) starting at row 5, and Column B contained Flute values (B) starting at row 5
- **Data range**: C5:Z10 contained the price values to be retrieved
- **Search criteria**: Located in cells AB4 (Grade header), AC4 (qty code), AD4 (Flute), and AE4 (Grade vertical)

The Python/openpyxl approach involved:
1. Reading the search criteria from cells AB4, AC4, AD4, and AE4
2. Reading vertical lookup values from columns A and B (rows 5-10)
3. Reading horizontal lookup values from rows 3 and 4 (columns C through Z)
4. Implementing nested matching logic to find the row index matching AE4 in column A AND AD4 in column B
5. Finding the column index matching AC4 in row 3 AND AB4 in row 4
6. Retrieving the value from the data range C5:Z10 at the matched row and column intersection
7. Writing the result to cell AE5

The formula required matching two vertical criteria simultaneously and two horizontal criteria simultaneously, which standard INDEX/MATCH syntax doesn't directly support without array formulas or helper columns.
